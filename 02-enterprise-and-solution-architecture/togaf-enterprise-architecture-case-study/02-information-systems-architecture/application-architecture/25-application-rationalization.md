# Application Rationalization

## Purpose
Decide which applications to keep, modernize, replace, or retire.

## Rationalization Categories
- Keep – High value, strategic  
- Modernize – Valuable but outdated  
- Replace – Redundant or limited  
- Retire – Low value or obsolete  

## Criteria
- Business value  
- Cost  
- Technical health  
- Strategic fit  

## Expected Outcomes
- Reduced application count  
- Lower operating cost  
- Clear application roadmap
