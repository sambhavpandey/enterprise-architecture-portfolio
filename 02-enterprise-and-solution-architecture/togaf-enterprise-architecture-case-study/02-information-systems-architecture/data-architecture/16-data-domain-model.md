# Data Domain Model

## Purpose
Define the core business data domains that support enterprise operations and decision making.

## Core Data Domains
- Customer  
- Product  
- Order  
- Supplier  
- Inventory  
- Finance  
- Employee  

## Domain Relationships
- Customer places Order  
- Order contains Product  
- Product uses Supplier  
- Supplier provides Inventory  
- Finance tracks Order and Supplier payments  
- Employee manages operations  

## Ownership
- Each domain has a business owner and data steward.  

## Objectives
- Single source of truth  
- Clear ownership  
- Improved data quality
